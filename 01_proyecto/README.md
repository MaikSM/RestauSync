# 01_proyecto

Esta carpeta contiene toda la documentación y desarrollo del proyecto RestauSync organizado por trimestres académicos.

## Estructura

### trimestre_2/
Contiene la fase inicial del proyecto:
- **01_planteamiento_general/**: Documentos de planteamiento del proyecto
- **02_mapa_de_procesos/**: Diagramas BPMN y mapas de procesos en PDF
- **03_recoleccion_informacion/**: Entrevistas y recopilación de requisitos
- **04_requerimientos/**: Especificaciones funcionales y no funcionales
- **05_casos_de_uso/**: Casos de uso del sistema
- **06_mockup/**: Prototipos visuales del sistema
- **07_ficha_tecnica/**: Cotización y ficha técnica del proyecto

### trimestre_3/
Fase de diseño y prototipos:
- **1_modelo_relacional/**: Modelo entidad-relación de la base de datos
- **2_normalización/**: Normalización de la BD a 3NF
- **3_diccionario_de_datos/**: Definición completa de tablas y campos
- **4_diagrama_de_clases/**: Diagramas UML de clases
- **5_diagrama_de_distribución/**: Arquitectura de distribución
- **6_prototipo_navegable/**: Prototipo funcional en HTML/CSS/JS con módulos para gestión de usuarios, inventario, pedidos, etc.

### trimestre_4/
Implementación final:
- **Base de datos/**: Scripts SQL para creación, consultas y procedimientos
- **Diagrama de clases/**: Diagramas finales de clases
- **Proyecto/**: Aplicación web completa en Flask (Python)
- **Trabajo/**: Versiones adicionales del proyecto
- **Vistas/**: Interfaces y prototipos adicionales

### trimestre_5/
Fase de desarrollo adicional (en progreso).

## Tecnologías Utilizadas
- Documentación: Microsoft Office, Draw.io
- Diagramas: MySQL Workbench, Visual Paradigm
- Prototipos: HTML5, CSS3, JavaScript puro
- Backend Legacy: Python Flask

Esta carpeta sirve como repositorio histórico del desarrollo incremental del proyecto.
